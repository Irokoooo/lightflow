# LightFlow 功能设计说明

## 1. 产品定位

LightFlow 是一个面向飞书多维表格标注场景的插件工作台，核心目标是降低 CB / Reviewer / QC 在多记录、多字段、多语言场景下的阅读、翻译、对比和填写成本。

## 2. 当前功能总览

| 功能 | 如何实现 | 功能介绍 | 如何使用 | 当前状态 |
| --- | --- | --- | --- | --- |
| 多视图工作台 | 基于本地视图配置与字段排序渲染 | 针对不同角色提供不同阅读/填写布局 | 顶部视图栏切换视图 | 可用 |
| `reviewer` 聚合视图 | 通过同 `Prompt ID` 记录聚合 + 双栏布局渲染 | Reviewer 同时阅读题目、回复与两份 Contributor 作业 | 在顶部切换到 `reviewer` 视图 | 可用 |
| `质检视图` 聚合视图 | 通过同 `Prompt ID` 记录聚合 + 三栏布局渲染 | QC 同时查看 Contributor 与 Reviewer 作业，再填写 QC 字段 | 在顶部切换到 `质检视图` | 可用 |
| 速读翻译视图 | 使用轻量字段布局 + 翻译开关 | 适合快速过题和重点阅读文本字段 | 切换到 `速读翻译` | 可用 |
| 同 Prompt ID 聚合 | 通过表内扫描记录并按 `Prompt ID` 过滤 | 自动把同题相关记录组织到同一阅读区域 | 使用聚合视图时自动生效 | 可用 |
| 字段自动翻译 | 基于字段级翻译组件 + 语言提示 | 对指定文本字段自动展示译文，减少来回复制 | 在支持翻译的字段下方查看译文 | 部分可用，依赖翻译能力 |
| 手动翻译 | 基于统一翻译组件与目标语言选择器 | 支持把任意字段内容或手动输入文本翻译到目标语言 | 字段下点击翻译，或切到“手动输入” | 可用 |
| 中文反馈转英文 | 手动输入模式 + 一键写回字段 | QC 可先用中文起草，再翻成英文后回填 | 在翻译组件中输入中文并点击“一键填入字段” | 可用 |
| Prompt ID 语种识别 | 基于 `Prompt ID` 前缀映射语言 | 为 BR / MX / PH / ID 题目提供更稳定的源语言判断 | 聚合视图自动生效 | 可用 |
| 评论字段英文直译 | 对 `Response 1 comments / Response 2 comments` 固定按英文处理 | 避免 CB 评论跟随题目语种误判 | 聚合视图自动生效 | 可用 |
| Markdown / LaTeX 渲染 | 使用 Markdown 渲染器和 KaTeX | 让回复、译文、公式内容更可读 | 点击顶部渲染开关开启/关闭 | 可用 |
| 内嵌滚动卡片 | 通过字段卡片和板块级滚动容器样式控制 | 避免长文本把整页撑得过长 | 在聚合视图中自动生效 | 可用 |
| 作业板块内嵌滚动开关 | 本地状态持久化 + 聚合作业列样式切换 | 允许整列 Contributor / Reviewer 作业作为一个滚动板块 | 聚合视图顶部勾选/取消 `作业板块内嵌滚动` | 可用 |
| 视图布局编辑 | 基于拖拽排序与本地配置保存 | 用户可直接调整当前可见字段的顺序、隐藏、翻译开关 | 点击顶部 `编辑布局` | 可用 |
| 翻译字段可配置 | 视图设置中保存 `translationFieldIds` | 用户可决定哪些字段启用翻译 | 在布局编辑中切换“翻译开/关” | 可用 |
| 字段编辑权限高亮 | 调用飞书单元格可编辑权限接口 | 区分当前字段是否可编辑 | 在字段卡片右上角查看“可编辑/只读” | 可用 |
| 仪表盘待做统计 | 基于“我是谁 + 完成字段”规则统计 | 帮助作业人看到自己的待做量和完成量 | 顶部仪表盘中配置规则 | 可用 |
| 下一条待做 | 根据仪表盘规则查找下一条未完成记录 | 减少手动找题 | 点击仪表盘 `下一条待做` | 可用 |
| 附件预览 | 对 PDF / CSV / DOC 等附件做针对性预览 | 提升附件字段可读性 | 在附件字段内直接展开查看 | 可用，个别格式仍可继续优化 |

## 3. 主要视图说明

### 3.1 `reviewer`

- 适用角色：Reviewer
- 主要目的：集中阅读题目与模型回复，并对比两份 Contributor 作业
- 主要区域：
  - 题目与回复阅读区
  - Contributor 1 作业字段
  - Contributor 2 作业字段
  - Reviewer 需要填写的字段

### 3.2 `质检视图`

- 适用角色：QC
- 主要目的：在同一页查看 Contributor 与 Reviewer 作业，并完成 QC 质检填写
- 主要区域：
  - 题目与回复阅读区
  - Contributor 1 作业字段
  - Contributor 2 作业字段
  - Reviewer 作业字段
  - QC 需要填写的字段

### 3.3 `速读翻译`

- 适用角色：需要快速阅读文本的人
- 主要目的：尽量减少布局干扰，快速看原文、译文和重点字段

## 4. 翻译设计说明

### 4.1 自动翻译规则

- `prompt / History / Response1 / Response2 / Likert justification` 可按配置启用翻译
- 聚合视图中的 `Response 1 comments / Response 2 comments` 默认自动翻译
- 评论字段默认按英文处理，不跟随 `Prompt ID` 推断语种

### 4.2 语言识别规则

- `BR` -> `pt-BR`
- `MX` -> `es-MX`
- `PH` -> `tl`
- `ID` -> `id`

### 4.3 使用方式

1. 打开带翻译的字段卡片
2. 检查源语言和目标语言
3. 等待自动翻译，或手动点击“翻译”
4. 如需回填，在手动输入模式中使用“一键填入字段”

### 4.4 当前限制

- 自动翻译是否可用，依赖当前环境能否拿到飞书翻译能力
- 如果环境没有飞书翻译授权，自动翻译不会继续消耗免费兜底接口额度
- 手动翻译仍可能走免费兜底接口，因此可能遇到额度限制

## 5. 布局与阅读体验

- 长文本字段支持内嵌滚动阅读
- 聚合作业列支持整列作为一个滚动板块
- Markdown / LaTeX 可以按需开启或关闭
- 用户可直接编辑视图布局、隐藏字段、开启或关闭字段翻译

## 6. 仪表盘能力

- 支持配置“我是谁”
- 支持配置“哪些条件算分给我”
- 支持配置“哪个字段算完成”
- 自动统计待做数量、已完成数量、预计剩余时间
- 支持跳转到下一条待做记录

## 7. 当前已知限制

### 7.1 加载速度

- 聚合视图为了匹配同 `Prompt ID`，需要扫描表内记录
- 切换记录时仍需要重新读取当前记录、字段元信息和部分权限信息
- 大表、字段多、聚合视图场景下，仍然会感受到加载偏慢

### 7.2 翻译能力

- 当前自动翻译依赖飞书环境能力
- 免费兜底翻译接口不适合持续自动批量翻译

### 7.3 手册助手

- 未来迭代方向：作业手册 AI 助手
- 当前状态：**暂不可用**
- 原因：当前插件环境中 `window.lark.ai` 与手册正文读取所需能力未稳定可用，无法完成“接入手册正文 + 引用式问答”链路
